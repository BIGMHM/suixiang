&#12288;&#12288;在本系列的前一篇，介绍了“<a href="../../2014/08/privacy-protection-8.md">流氓的桌面软件有哪些替代品</a>”。今天主要介绍一下，如何对付那些“<b>缺乏替代品</b>的流氓软件”（比如 QQ 和迅雷）。<br />&#12288;&#12288;本文会逐一介绍5种不同的方案，供大伙儿参考。在本文末尾会总结每种方案分别适用哪些情况。<a name='more'></a><!--program-think--><br /><br /><h2>★方案1：用流氓软件的“Web 版本”进行替代</h2><br /><h3>◇概述</h3><br />&#12288;&#12288;某些流氓软件的开发商，除了开发桌面客户端版本，有时候也会提供“Web 版本”。在这种情况下，咱们就可以拿“Web 版本”来取代桌面版本。<br /><br /><h3>◇举例</h3><br />&#12288;&#12288;比如：疼逊公司提供了一个 Web QQ。那么你就可以用 Web QQ 来替代桌面版的 QQ；<br />&#12288;&#12288;比如：很多网盘提供商同时提供网盘的“Web 操作界面”和“桌面客户端”。那么你就可以：只用 Web 界面操作网盘，不安装桌面客户端。<br /><br /><h3>◇不同“Web 应用”的安全性等级</h3><br />&#12288;&#12288;基于 Web 的应用软件，有好几种不同的类型，在隐私方面的安全性也各不相同。下面逐一介绍。在下面的介绍中，会提及“浏览器扩展”和“浏览器插件”。如果你搞不清楚这两者的差别，先看之前的“<a href="../../2012/08/howto-prevent-hacker-attack-5.md">这篇博文</a>”，其中有一段文字是“◇插件和扩展的区别”。<br /><br />&#12288;&#12288;<b>纯 Web</b><br />&#12288;&#12288;所谓的“纯 Web”，就是说只用到 HTML、JavaScript脚本、CSS 样式。除此之外，没有用到其它其它东西。<br />&#12288;&#12288;这种形式的 Web 应用是最安全的。对于这种形式，JS 脚本运行在浏览器内置的脚本沙箱中，它对文件系统的访问是受限的。换句话说，【无法】随意操作电脑本地的文件系统。<br /><br />&#12288;&#12288;<b>使用了“扩展/extension”技术</b><br />&#12288;&#12288;浏览器扩展能干的事情，比“纯 Web”更多。多出来的功能主要是：能够操作本地的文件系统。假设你安装了一个不怀好意的扩展，理论上它是可以读取你本地的某些敏感文件并通过 Web 方式（HTTP方式）发送出去。<br />&#12288;&#12288;提醒一下：并不是所有的浏览器都支持“扩展”机制。<br /><br />&#12288;&#12288;<b>使用了“插件/plugin”技术</b><br />&#12288;&#12288;“插件”比“扩展”牛逼的地方在于，插件是本地代码（也叫“原生代码”）。也就是说，普通客户端软件能干的事情，插件都能干。所以，插件的危险性更大，基本上等同于客户端软件。<br />&#12288;&#12288;比如前几天曝光的“支付宝偷偷监控网络流量”。那个监控你网络流量的进程，就是在安装支付宝控件之后，偷偷带进去的。而“支付宝控件”本质上就是“浏览器插件”。<br /><br />&#12288;&#12288;综上所述，如果你企图用“Web 版本”来替代“桌面版本”，至少先要确保“Web 版本”没有使用“浏览器插件”技术。用到了“插件”的 Web 应用，其隐私威胁跟桌面客户端类似――也就没有必要替代了。<br /><br /><h3>◇优点</h3><br />&#12288;&#12288;主要优点是：操作简单，无需进行各种额外的配置，无需安装额外的桌面软件。<br /><br /><h3>◇缺点</h3><br />&#12288;&#12288;此招数的缺点有两个：<br />&#12288;&#12288;<b>缺点1</b><br />&#12288;&#12288;使用了 Web 版本之后，虽然可以避免桌面客户端偷窥你的本地文件，但是你通过 Web 端产生的内容还是会被 Web 服务器收集到。<br />&#12288;&#12288;比如你用了 Web QQ 之后，虽然它无法偷窥你的本地文件，但是腾讯服务器依然可以偷窥你的聊天内容。<br /><br />&#12288;&#12288;<b>缺点2</b><br />&#12288;&#12288;通常，Web 版本的功能不如桌面版本的功能那么丰富。<br />&#12288;&#12288;比如“Web QQ”的功能比“桌面 QQ”的功能少很多。<br /><br /><h2>★方案2：使用“网络防火墙”限制联网</h2><br /><h3>◇概述</h3><br />&#12288;&#12288;如果某个流氓软件的<b>基本功能跟网络无关</b>，那么你可以采用网络防火墙限制该软件的网络行为。<br /><br /><h3>◇举例</h3><br />&#12288;&#12288;比如：你需要用国产的播放器来播放【本地的】视频文件和音乐文件。并且你既不需要“自动同步字幕”，也不需要“自动同步歌词”之类的功能。在这种情况下，该软件就完全没有理由访问网络。那么你就可以通过配置网络防火墙，让该软件无法联网。<br /><br /><h3>◇网络防火墙的局限性</h3><br />&#12288;&#12288;<b>局限性1</b><br />&#12288;&#12288;某些流氓软件需要以管理员权限安装。在这种情况下，流氓软件也就获得了管理员权限。在“管理员权限”下，几乎可以进行任何软件层面操作。从理论上讲，可以利用管理员权限进行某些操作，来绕过网络防火墙的监控。<br /><br />&#12288;&#12288;<b>局限性2</b><br />&#12288;&#12288;某些比较狡猾的流氓软件，自己的主程序不会直接联网，而是通过创建一个临时的 exe 文件并启动该 exe（子进程），让子进程完成敏感的联网操作。以此来骗过网络防火墙。对这类软件，即使你禁止该软件的主程序联网，它还是能通过启动子进程的方式，把某些敏感的资料偷偷发送出去。<br /><br /><h3>◇优点</h3><br />&#12288;&#12288;用此招数，你可以直接使用流氓软件的桌面版本。相比“方案1”的优势在于――能保留软件的全部功能。<br />&#12288;&#12288;另外，此招数可以跟下面要介绍的“方案3”搭配组合。<br /><br /><h3>◇缺点</h3><br />&#12288;&#12288;此招数的缺点包括：<br />&#12288;&#12288;<b>缺点1</b><br />&#12288;&#12288;如果该流氓软件的基本功能依赖于网络（比如：聊天工具、下载工具），就没法用这招。<br /><br />&#12288;&#12288;<b>缺点2</b><br />&#12288;&#12288;网络防火墙的限制有可能被绕过（参见上述提到的两个局限性）<br /><br /><h2>★方案3：用流氓软件的“绿色版本”进行替代</h2><br /><h3>◇概述</h3><br />&#12288;&#12288;某些流氓软件可以制作成“绿色版本”（俗称“绿化”）。绿化之后的软件，无需安装，而且也无需管理员权限就可以运行。因此，你可以创建一个“低权限的用户”，以这个用户的身份来运行该流氓软件。关于 Windows 的“用户组、用户权限”等概念的更多介绍，请看之前的“<a href="../../2010/06/howto-prevent-hacker-attack-1.md">这篇博文</a>”。<br />&#12288;&#12288;另外，从 WinXP 开始，Windows 内置了“快速用户切换”（洋文叫：Fast User Switching）的功能。你可以利用此功能在不同的用户之前切换。不想使用“用户切换”的同学，也可以在同一个桌面上运行“隶属于不同用户的进程”（具体做法，上面那篇博文也有提到）。<br /><br /><h3>◇需要结合“文件系统访问控制（ACL）”</h3><br />&#12288;&#12288;和这个招数搭配的措施是：你要适当地配置文件系统的访问控制权限。比如你日常操作电脑的是“A用户”，用来运行流氓软件的低权限用户是“B用户”。那么你需要设置那些存放敏感文件的目录，让“B用户”【无法读写】这些目录。至于如何设置目录的 ACL，稍微 Google 一下就能查到教程。<br />&#12288;&#12288;提醒一下：Windows 支持的文件系统主要有两类：NTFS 和 FAT 系列（FAT12、FAT16、FAT32）。“FAT 系列”【不支持】访问控制权限。<br /><br /><h3>◇举例</h3><br />&#12288;&#12288;比如“迅雷”是可以制作成绿色版本的。俺分别在它的5.8版本和7.2版本试验过。<br /><br /><h3>◇优点</h3><br />&#12288;&#12288;用此招数，你可以直接使用流氓软件的桌面版本。相比“方案1”的优势在于――能保留软件的全部功能。<br />&#12288;&#12288;另外，此招数可以跟“方案2”搭配组合。<br /><br /><h3>◇缺点</h3><br />&#12288;&#12288;此招数的缺点包括：<br />&#12288;&#12288;<b>缺点1</b><br />&#12288;&#12288;如果某个流氓软件无法制作绿色版本，这招就失效了。<br /><br />&#12288;&#12288;<b>缺点2</b><br />&#12288;&#12288;操作相对繁琐（你需要先制作绿色版本，然后还需要创建专门的用户，最后还要设置文件系统的 ACL）。<br /><br /><h2>★方案4：使用“沙箱软件”进行隔离</h2><br /><h3>◇概述</h3><br />&#12288;&#12288;有一类安全工具叫做“沙箱”（有时也称“沙盒”）。沙箱软件可以创建出一个独立的运行环境，以实现【进程级别】的隔离。<br />&#12288;&#12288;所谓“进程级别的隔离”就是说：你可以在沙箱中运行某个进程，沙箱软件负责把该进程跟系统的其它部分隔离开来。即使该进程图谋不轨，想要进行某些危险操作，这些操作也被限制在沙箱之内。<br />&#12288;&#12288;所以，咱们可以利用“沙箱软件”来运行那些流氓软件。<br /><br /><h3>◇举例</h3><br />&#12288;&#12288;目前，【国外】比较有名的沙箱软件是 <a href="https://en.wikipedia.org/wiki/Sandboxie" target="_blank" rel="nofollow">Sandboxie</a>。如果你担心网页挂马，可以让浏览器运行在 Sandboxie 的沙箱之内。<br /><br /><h3>◇沙箱的局限性</h3><br />&#12288;&#12288;<b>局限性1</b><br />&#12288;&#12288;沙箱软件依赖于操作系统本身的一些机制（比如系统的 hook 机制）。如果操作系统本身出现一些安全漏洞，可能会导致某些流氓软件突破沙箱的隔离。<br />&#12288;&#12288;另外，如果沙箱软件本身考虑不周到，也会导致隔离环境出现漏洞，可能被流氓软件用来突破沙箱。<br /><br />&#12288;&#12288;<b>局限性2</b><br />&#12288;&#12288;有些沙箱软件只能对磁盘操作进行隔离，但是【无法】对网络操作进行隔离（无法禁止流氓软件联网）。<br />&#12288;&#12288;如果你需要隔离网络操作，但是你用的沙箱又没有此功能，那么你可以考虑把此招数跟“方案2”搭配。<br /><br /><h3>◇优点</h3><br />&#12288;&#12288;用此招数，你可以直接使用流氓软件的桌面版本。相比“方案1”的优势在于――能保留软件的全部功能。<br />&#12288;&#12288;不管流氓软件是否能制作绿色版本，都可以使用此招数。所以此招数比“方案3”更灵活。<br /><br /><h3>◇缺点</h3><br />&#12288;&#12288;主要缺点有如下几个：<br />&#12288;&#12288;<b>缺点1</b><br />&#12288;&#12288;某些软件（尤其是比较底层的软件）运行在沙箱中会出现兼容性问题。<br /><br />&#12288;&#12288;<b>缺点2</b><br />&#12288;&#12288;由于前面提到的“沙箱的局限性”，会导致隔离不彻底。<br /><br />&#12288;&#12288;<b>缺点3</b><br />&#12288;&#12288;多数沙箱软件通常无法跨系统（比如前面提到的 Sandboxie 是运行在 Windows 上，所以它只能用来隔离 Windows 软件）。<br />&#12288;&#12288;如果你对跨平台的需求不明显，这个缺点基本可以忽略。<br /><br /><h2>★方案5：使用“操作系统虚拟机”进行隔离</h2><br /><h3>◇概述</h3><br />&#12288;&#12288;所谓的“操作系统虚拟机”（为了打字省力，以下简称“VM”），就是用专门的软件在你现有的操作系统内部再虚拟出若干个操作系统。在这种情况下，你原有的操作系统称为“物理系统”或“Host OS”；那些虚拟出来的系统称为“虚拟系统”或“Guest OS”。<br />&#12288;&#12288;有了 VM 之后，你就可以创建出一个或多个 Guest OS，在那上面运行流氓软件。由于 Guest OS 与 Host OS 是彻底隔离的，你完全不用担心 Guest OS 里面的进程偷窥 Host OS 的文件系统。<br />&#12288;&#12288;所谓的“彻底隔离”，俺补充说明一下：<br />&#12288;&#12288;因为 Guest OS 运行于 Host OS 之上，Host OS 是可以看到 Guest OS 里面的东西；反之则【不行】。如果有多个 Guest OS，互相之间看不到。<br />&#12288;&#12288;关于 VM 的更多介绍，请参见《<a href="../../2012/10/system-vm-0.md">扫盲操作系统虚拟机</a>》系列教程。<br /><br /><h3>◇举例</h3><br />&#12288;&#12288;至少在10年前，俺就以虚拟系统（Guest OS）作为自己日常的使用系统。俺的 Host OS 除了装一个虚拟机软件，其它软件几乎都【不】装。需要用到的软件都装在 Guest OS 里面。而且俺的笔记本电脑里面装了 N 多的虚拟机，分别派不同的用处。比如：<br />一个专门用于“编程随想”这个身份<br />一个专门用于公司的办公环境（写文档、收发邮件）<br />一个专门用于公司的开发环境（留个开环境帮同事解决疑难编程问题）<br />一个专门用于业余时间的开发环境（业余时写代码）<br />一个专门用于业余时间的上网环境（业余时上网）<br />（还有很多，不一一列举）<br />&#12288;&#12288;搞这么多虚拟机的好处在于，把自己的敏感数据分散开――即便某个虚拟系统被入侵，其它虚拟系统【不受】影响。<br />&#12288;&#12288;而且 VM 软件都会提供【快照功能】――这个功能是 VM 的精华。比如每次刚装好系统，或者刚升级好系统，俺都会做一个快照。用了几天之后，就回退到这个快照。就算这几天当中被恶意软件（病毒/木马/流氓软件）污染了系统，只要回退到快照，虚拟系统又纯洁如初 :)<br />&#12288;&#12288;前几天有读者问俺用的是哪个系统清理工具。俺回答说：有了 VM 软件，根本【无需】系统清理工具。为啥捏？因为每次回退快照，系统垃圾自动就没了（等同于一次彻底的清理）。<br /><br /><h3>◇优点</h3><br />&#12288;&#12288;VM 的优点很多，至少包括如下：<br />&#12288;&#12288;<b>优点1</b><br />&#12288;&#12288;最好的兼容性。<br />&#12288;&#12288;前面提到的“方案3”和“方案4”，都可能会导致一些兼容性的问题。而 VM 完全没有。<br /><br />&#12288;&#12288;<b>优点2</b><br />&#12288;&#12288;最好的隔离性（关于这点，下面俺单独开了一个章节详述）<br /><br />&#12288;&#12288;<b>优点3</b><br />&#12288;&#12288;VM 的快照功能可以帮你完成很多额外的工作（参见俺刚才的举例）。<br /><br />&#12288;&#12288;<b>优点4</b><br />&#12288;&#12288;彻底实现“跨平台”。<br />&#12288;&#12288;比如你可以用 Windows 做 Host OS，然后在其中虚拟出一个 Linux 或 Mac 的 Guest OS；反之亦然。而沙箱软件做不到这点。<br /><br /><h3>◇缺点</h3><br />&#12288;&#12288;俺个人认为：对于那些安全要求高的网友，VM 的方案是最佳方案。此方案有如下两个不太明显的缺点：<br />&#12288;&#12288;<b>缺点1</b><br />&#12288;&#12288;如果要使用，你的硬件配置不能太低。<br />&#12288;&#12288;如今电脑硬件的配置已经越来越好（动不动就是好几 GB 的内存，动不动就 4核/8核），这个缺点基本可以忽略。<br /><br />&#12288;&#12288;<b>缺点2</b><br />&#12288;&#12288;要花点时间学习如何使用 VM。<br />&#12288;&#12288;不过这个缺点也好办――俺在2年前（2012）就写了一个《<a href="../../2012/10/system-vm-0.md">扫盲操作系统虚拟机</a>》的系列教程。不会用 VM 的同学可以先去看这个教程。<br /><br /><h2>★为啥“OS虚拟机”的隔离性比“沙箱”更好？</h2><br />&#12288;&#12288;俺在博客上写了不少操作系统虚拟机的博文。但是一直没有介绍过“沙箱软件”。曾经有几个读者在留言中问道：为啥从来没普及过沙箱？今天借着这个机会说一下。<br />&#12288;&#12288;前面俺提到了 VM 的各种优点。这里重点想说其中一点――<b>隔离性</b>。为啥捏？如果你对安全的要求比较高，“隔离性”是关键。而“VM”的隔离性远远好于“沙箱”。具体的分析如下：<br /><br /><h3>◇关于隔离的“维度”</h3><br />&#12288;&#12288;不论是 VM 还是沙箱，说白了都是为了创造出一个隔离的软件环境。这时候要考虑的维度有很多，至少包括：文件系统、内存、网络、外设（比如 USB 口）。<br />&#12288;&#12288;主流的 VM 软件可以支持上述【所有的】维度；而大部分沙箱软件只能做到文件系统和内存的隔离。有些沙箱虽然也支持网络隔离，但是其网络隔离的功能不如 VM 那么丰富（主流的 VM 软件通常支持至少3种以上的虚拟网卡模式）。<br /><br /><h3>◇关于操作系统本身的漏洞</h3><br />&#12288;&#12288;在使用“沙箱”的情况下，沙箱软件和流氓软件都运行在同一个操作系统中。如果这个操作系统出现某些安全漏洞，【可能会】导致流氓软件突破沙箱的隔离边界。<br />&#12288;&#12288;相对而言，使用 VM 的情况下，同时存在两个操作系统。VM 软件本身是运行在 Host OS 之上，而流氓软件运行在 Guest OS 之上。由于 VM 本身并不依赖 Guest OS。所以 Guest OS 出现安全漏洞，【不太可能】导致隔离边界被突破。<br />&#12288;&#12288;有些爱思考的同学会问了：如果 Host OS 出现安全漏洞，是否有可能导致隔离边界被突破。这种可能性是有的，但是操作的难度很大。为啥捏？因为流氓软件运行在 Guest OS 之内，它根本看不到 Host OS。换句话说，流氓软件根本就无法知道 Host OS 是啥类型的。连操作系统的类型都不知道，如何去利用操作系统的漏洞？<br /><br /><h2>★总结</h2><br />&#12288;&#12288;在本文的结尾，稍微做一下总结发言，说一下每种招数适用的情况。<br /><br /><h3>◇方案1――用“Web 版本”替代</h3><br />&#12288;&#12288;前提：对应的流氓软件提供 Web 版本<br />&#12288;&#12288;本方案无需额外的配置，也无需再安装其它桌面软件。所以适合那些非常菜鸟的网友，或者是那些非常懒的网友。另外，如果你只是想临时用一下某个流氓软件，也可以用这个招数。<br /><br /><h3>◇方案2――用“网络防火墙”限制联网</h3><br />&#12288;&#12288;前提：对应的流氓软件的基本功能跟网络无关（比如输入法、播放本地视频/音乐文件）<br />&#12288;&#12288;除了需要配置网络防火墙，无需额外安装其它软件。适合那些比较菜鸟的同学或者是比较懒的同学。<br /><br /><h3>◇方案3――用“绿色版本”替代</h3><br />&#12288;&#12288;前提：对应的流氓软件能够“绿化”<br />&#12288;&#12288;需要自己手工制作绿色版本，而且还需要配置额外的操作系统用户和文件系统的访问控制权限。所以此招数适合于具有一定折腾能力的网友。而且电脑硬件又很差，用不了虚拟机方案。<br /><br /><h3>◇方案4――用“沙箱软件”隔离</h3><br />&#12288;&#12288;前提：流氓软件需要能跟沙箱软件兼容。<br />&#12288;&#12288;如果你电脑硬件太差，用不了虚拟机。而且你又搞不定绿色版本（比如流氓软件不支持“绿化”，或者你不懂得如何“绿化”），那么你可以用此方案。<br /><br /><h3>◇方案5――用“操作系统虚拟机”隔离</h3><br />&#12288;&#12288;前提：无任何限制。<br />&#12288;&#12288;此方案堪称终极解决方案，尤其适合于对安全要求比较高的同学。此方案需要有一定的折腾能力（不需很高），而且电脑的硬件配置至少是中等水平。<br /><br /><h3>◇所有这五种方案的局限性</h3><br />&#12288;&#12288;不论你用的是哪个方案。如果你想使用某个流氓软件，并且<b>该流氓软件的基本功能是依赖于网络的</b>，那么，这款流氓软件就有可能收集到你的某些隐私并发送给厂商的服务器。即使你用了最安全的“虚拟机隔离”，顶多也只是降低了泄漏的信息量而已。<br />&#12288;&#12288;举例来说：你在虚拟机里面运行 QQ，可以阻止 QQ 偷偷扫盲你的 Host OS 的磁盘。但是，你的所有聊天内容，腾讯的服务器上都有记录。所以，<b>最安全的是：既不用流氓公司的软件，也不用流氓公司的网络服务</b>。<br /><br /><a href="../../2013/06/privacy-protection-0.md#index">回到本系列的目录</a><div class="blogger-post-footer">
