&#12288;&#12288;前2个月写了一篇《<a href="../../2013/12/create-bootable-usb-stick-from-iso.md">如何用ISO镜像制作U盘安装盘（通用方法、无需WinPE）</a>》，收到很多读者的反馈。有几个读者抱怨自己的电脑太旧，不支持“U盘启动”。他们希望俺介绍一下用“内置硬盘”的方式安装新系统。这个周末俺折腾了一下，今天分享一下折腾的心得。<a name='more'></a><!--program-think--><br /><br /><h2>★本文适用的硬件环境</h2><br />&#12288;&#12288;首先说一下本文适用的环境――符合如下两个条件之一，可以考虑用本文介绍的方法。<br />1、电脑没有光驱 and 主板不支持“USB启动”<br />2、手头没有物理光盘 and 没有U盘 and 没有外置移动硬盘<br /><br /><h2>★全新安装 VS 升级安装</h2><br />&#12288;&#12288;本文介绍的方法，强调是【全新安装】。所谓的“全新安装”有别于“升级安装”。“升级安装”有如下缺点：<br />1. 有些系统无法进行升级安装（比如无法从 XP 升级到 2003）<br />2. 无法进行“降级”（比如原有系统是 Win7，无法降为 WinXP）<br />3. 使用“升级安装”得到的系统不够“纯洁”（比如原有系统中的病毒或流氓软件会继续遗留在新系统中）<br />&#12288;&#12288;综上所述，俺通常不建议用“升级安装”。所以，今天不讨论“升级安全”的话题。<br /><br /><h2>★“全新安装”的两种模式</h2><br /><h3>◇概述</h3><br />&#12288;&#12288;用【内置硬盘】进行“全新安装”，可以有两种模式：<br />&#12288;&#12288;<b>“直接运行”模式</b><br />&#12288;&#12288;直接把新系统的安装盘解压缩到本地硬盘的某个目录，然后直接运行其中的 setup.exe 开启安装过程。<br />&#12288;&#12288;<b>“重启动”模式</b><br />&#12288;&#12288;先用某种方式设置当前系统的启动项，然后通过“重启动”进入安装环境。<br /><br /><h3>◇优缺点对比</h3><br />&#12288;&#12288;“直接运行”模式具有如下缺点：<br />&#12288;&#12288;缺点1：无法降级<br />&#12288;&#12288;在“直接运行”的模式下，如果你要安装的【新】系统，版本号比现有系统更小，安装程序会拒绝安装。<br />&#12288;&#12288;缺点2：无法跨“32位/64位”<br />&#12288;&#12288;在“直接运行”的模式下，如果现有的系统是32位，就无法直接运行64位系统的 setup.exe。<br /><br /><h3>◇小结</h3><br />&#12288;&#12288;因为存在上述缺点，所以本文只介绍第二种模式――“重启动模式”。<br /><br /><h2>★当前主流 Windows 的两个系列――扫盲 NT 版本号</h2><br />&#12288;&#12288;在进入正题之前，俺先卖弄一个技术常识――关于 NT 的版本号。<br />&#12288;&#12288;如今主流的 Windows，不管其产品名如何变幻，其内核都是 Windows NT。NT 是洋文“新技术”的缩写，以区别于更落后的 Win95/Win98/WinME（90后的同学估计没接触过这些老系统）。<br />&#12288;&#12288;NT 的发展大致经历了4代，具体如下：<br /><pre><font face="Courier New">----------------<br />Windows NT 3.1          版本号 3.1（为啥第一代不是 1.0？因为要跟当时 DOS 上运行的 Windows 3.1 保持版本号一致）<br />Windows NT 3.5          版本号 3.5<br />Windows NT 3.51         版本号 3.51<br />----------------<br />Windows NT 4.0          版本号 4.0<br />----------------<br />Windows 2000            版本号 5.0<br />Windows XP              版本号 5.1<br />Windows Server 2003     版本号 5.2<br />----------------<br />Windows Vista           版本号 6.0<br />Windows Server 2008     版本号 6.0<br />Windows 7               版本号 6.1<br />Windows Server 2008 R2  版本号 6.1<br />Windows 8               版本号 6.2<br />----------------</font></pre>&#12288;&#12288;NT3 和 NT4，如今基本不用了；从 Win2K 一直到 Win2K3，都称为“NT5”；从 Vista 开始一直到 Win8.1，称为“NT6”。NT5/NT6 两个系列，差异是很大滴。这两天，俺企图找到某种“通杀”的硬盘安装方法――能同时搞定 NT5 和 NT6――可惜没能如愿 :( 所以捏，只好勉为其难，分别介绍两种“半通用”的方法，分别针对 NT5 和 NT6。这两种方法的操作完全不同。<br /><br /><h2>★针对 Windows 2000/XP/2003 的方法</h2><br />&#12288;&#12288;如果你不需要安装 2000/XP/2003，请直接跳过本章节，直接到下一章节（以免浪费时间）。<br /><br /><h3>◇原理说明</h3><br />&#12288;&#12288;先依靠 grub4dos 这个开源的小工具，在当前系统中加出新的启动项，然后再配合 DOS镜像，就可以启动出一个 DOS环境。所有 NT5 系列的 Windows 都支持在 DOS 下运行安装程序。如此一来，就可以用本地硬盘进行“全新安装”。<br /><br /><h3>◇和其它方法的对比</h3><br />&#12288;&#12288;<b>对比“直接运行模式”</b><br />&#12288;&#12288;直接解压缩 ISO 然后运行 setup 的方式（也就是刚才提到的“直接运行模式”）存在若干缺点，前面已经分析了。<br /><br />&#12288;&#12288;<b>对比“用 grub直接启动 ISO”</b><br />&#12288;&#12288;还有一种类似的安装方法，也是用 grub4dos 来添加启动项，然后让 grub4dos 直接启动 ISO 镜像。这种方法的缺点是：<br />缺点1：兼容性不够好<br />比方说：假如 ISO 文件在硬盘上【没有】连续存放，grub4dos 直接启动这个 ISO 会失败。虽然 grub4dos 有一个 --mem 的参数可以把 ISO 镜像载入内存再启动（以此来解决“连续存放”的问题），但对于小内存的电脑，这招显然不合适。<br />缺点2：灵活性不够好<br />因为 Windows 安装程序自带的硬盘分区功能比如弱。而如果用 grub4dos 先启动出 DOS 环境，就可以用 DOS 下专用的分区工具（比如 PQmagic）调整分区布局/分区格式。<br /><br />&#12288;&#12288;<b>对比“用 grub直接启动 ntldr”</b><br />&#12288;&#12288;这种方式比“grub直接启动 ISO”的兼容性好一些。但依然存在“灵活性不足”的缺点。<br /><br /><h3>◇步骤1：准备一个硬盘分区</h3><br />&#12288;&#12288;首先，你要准备一个硬盘分区，用来存放 Windows 的安装文件。该分区必须使用 FAT16 或 FAT32 文件系统。这个硬盘分区必须具有足够大的“空闲空间”。需要多大的“剩余空间”，主要看你要新装的是哪种 Windows。<br />&#12288;&#12288;为了叙述方面，下面俺称这个分区为“安装文件分区”。<br /><br /><h3>◇步骤2：安装 grub4dos</h3><br />&#12288;&#12288;先到它的官网下载 grub4dos 的软件包，文件名是“grub4dos-0.4.4.zip”（下载页面在“<a href="http://download.gna.org/grub4dos/" target="_blank" rel="nofollow">这里</a>”），接着下载 grubinst 软件包，文件名是“grubinst-1.1-bin-w32-2008-01-01.zip”（下载页面在“<a href="http://download.gna.org/grubutil/" target="_blank" rel="nofollow">这里</a>”）<br />&#12288;&#12288;把下载好的 grubinst 解压到某个目录，运行其中的 <b>grubinst_gui.exe</b>，参照如下截图进行设置。<br /><br /><center><img 请翻墙"></center><br />&#12288;&#12288;接着在 grub4dos 压缩包里面找一个名叫 <b>grldr</b> 的文件，把这个文件放到前面提到的那个“安装文件分区”的【根目录】下。<br /><br />&#12288;&#12288;完成上述步骤，grub4dos 就替代了原有硬盘的 MBR（主引导扇区）。<br /><br /><h3>◇步骤2：配置 grub4dos</h3><br />&#12288;&#12288;俺准备了一个现成的 grub4dos 配置文件，这个文件的文件名必须是 <b>menu.lst</b>，这个文件也跟 grldr 一样，放在“安装文件分区”的【根目录】下。<br />&#12288;&#12288;menu.lst 的内容如下：<br /><blockquote style="background-color:#DDD;">timeout 10<br />default 0<br /><br />title old system<br />chainloader <b style="color:red;">(hd0,0)</b>+1<br /><br />title install<br />chainloader <b command line<br />commandline<br /><br />title reboot<br />reboot</blockquote><br />&#12288;&#12288;“title old system”那一段对应你原有的系统，“title install”那一段对应“安装文件分区”，“title DOS”那一段对应 DOS镜像（等一下会说）。<br />&#12288;&#12288;一般情况下，你需要稍微修改一下 menu.lst 配置文件（因为你的硬盘分区跟俺的硬盘分区，可能有差异）。【可能】修改的地方有两处，俺分别标注了红色。<br />&#12288;&#12288;grub4dos 的配置文件采用如下方式表示硬盘分区：<br /><b>(hdX,Y)</b>  表示第X个物理硬盘的第Y个分区（X 和 Y 是以0开头的整数）。<br />X=0 表示第一个硬盘，X=1 表示第二个硬盘，以此类推。<br />Y的含义稍复杂，列表如下：<br />0  第1个主分区<br />1  第2个主分区<br />2  第3个主分区<br />3  第4个主分区<br />4  第1个逻辑分区<br />5  第2个逻辑分区<br />（以此类推）<br />&#12288;&#12288;稍微解释一下：对于传统的 MBR 分区布局，最多只能有4个“主分区”。为了让分区数超过4个，特地增加了“扩展分区”的功能，一个硬盘只能有一个扩展分区，扩展分区内部可以分“任意多个”逻辑分区。<br /><br /><h3>◇步骤3：准备一个 DOS 镜像</h3><br />&#12288;&#12288;如果你喜欢折腾装机，或许你自己已经有一个现成的 DOS 镜像。如果没有的话，自己去网上搜一个。<br />&#12288;&#12288;把你找来的 DOS 镜像放到“安装文件分区”的【根目录】下，文件名改成 <b>dos.img</b><br /><br /><h3>◇步骤4：测试一下 grub4dos</h3><br />&#12288;&#12288;经过前面几个步骤，grub4dos 应该准备妥当了。你最好先测试一下 grub4dos 是否能正常工作。<br /><h3>◇步骤5：准备安装文件</h3><br />&#12288;&#12288;然后你找来要安装的系统盘（<b>强调一下：必须是“微软官方镜像”</b>）。只要是 Win2000/XP/2003 的官方镜像，ISO 里面会有一个名叫“i386”的目录。你把这个目录从 ISO 里面解压缩到“安装文件分区”。<br />&#12288;&#12288;说到这里，顺便抹黑一下山寨的安装盘。在咱们天朝有不少山寨的安装盘，里面可谓是鱼龙混杂。某些低档的安装盘，充斥着流氓软件甚至木马。所以，（要装 Windows 的话）俺一贯推荐用微软原版的镜像，以确保系统的“纯洁”。<br /><br /><h3>◇步骤6：启动到 DOS</h3><br />&#12288;&#12288;重启系统，在 grub4dos Installer”这个小工具添加启动项，同时这个小工具还会完成相应的配套设置。重启动之后直接可以全新安装 NT6 系列。<br /><br /><h3>◇和其它方法的对比</h3><br />&#12288;&#12288;首先，直接解压缩 ISO 然后运行 setup 的方式（也就是刚才提到的“直接运行模式”）存在若干缺点，前面已经分析了。<br />&#12288;&#12288;对于 NT6 系列的硬盘安装，还有一个很流行的方法：先把 ISO 解压缩到某个分区的【根目录】，然后运行 /boot/bootsect.exe 进行设置，再把该分区设置为“活动”状态。然后就可以通过重启动的方式，启动该分区的安装程序。<br />&#12288;&#12288;这个方法虽然很简单，但有个缺点――“无法跨 32位/64位”。比如说：当前系统是 32位，而你想安装 64位 的 Win7或Win8。对于 64位 的Win7/Win8，ISO 镜像里面的“/boot/bootsect.exe”也是64位，无法在 32位 的系统中运行。<br /><br /><h3>◇步骤1：准备一个硬盘分区</h3><br />&#12288;&#12288;首先，你要准备一个硬盘分区，用来存放 Windows 的安装文件。这个硬盘分区必须具有足够大的“空闲空间”。然后把安装光盘的内容解压缩到这个分区的【<b>根目录</b>】下。<br />&#12288;&#12288;为了叙述方面，下面俺称这个分区为“安装文件分区”。<br /><br /><h3>◇步骤2：下载/运行 NT6 HDD Installer</h3><br />&#12288;&#12288;这个小工具的官网也就是作者自己的
